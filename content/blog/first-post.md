## 證明逐步檢查

**步驟 1:** $C(A) = C(A^T)$ 隱含相同的母空間 $\mathbb{R}^n$。

正確。$C(A) \subseteq \mathbb{R}^m$,$C(A^T) \subseteq \mathbb{R}^n$,要相等就必須在同一個母空間裡,所以 $m = n$,A 是方陣。

**步驟 2:** 設 $\text{rank}(A) = k$,則 $C(A)$ 和 $C(A^T)$ 都是 $\mathbb{R}^n$ 裡同一個 $k$ 維子空間。

正確,且你加的那句「(一定會)」很關鍵——你在強調 $\dim C(A) = \dim C(A^T) = k$ 是 rank 的對稱性,不是條件,而是定理。這個觀察值得被獨立記住:**任何矩陣的 row rank 永遠等於 column rank**,這是線代裡最不顯然但最深刻的事實之一。

**步驟 3:** $N(A) = C(A^T)^\perp$,$N(A^T) = C(A)^\perp$,而且正交補是唯一的。

正確。「正交補唯一」這個性質很關鍵——對給定母空間裡的任何子空間 $V$,$V^\perp$ 是被 $V$ 完全決定的、唯一的子空間。所以一旦 $C(A) = C(A^T)$,它們各自的正交補(就是兩個 null space)也必然相等。

**步驟 4:** 因此 $N(A) = N(A^T)$,與題目假設 $N(A) \neq N(A^T)$ 矛盾。

正確。所以這樣的 A 不存在。

整個證明從假設到矛盾,中間每一步都用了恰好一個明確的定理(rank 對稱性、正交補的定義、正交補的唯一性),沒有多餘的步驟,沒有偷渡的假設。這是一個**「合格的數學證明」**——比你前一版本(誤用對稱、誤用維度推集合)進步了一個層次。

### 為什麼「想像 entries」會失敗

當題目給的是 entries 等式($A = A^T$、$A_{ij} = 0$ 之類),想像 entries 是對的——你需要操作具體的數字。

但當題目給的是**子空間等式**($C(A) = C(A^T)$、$N(A) \subseteq C(A^T)$ 之類),entries 只是子空間性質的「載體」,不是性質本身。同一個子空間可以由無數不同的 entries 矩陣產生(任何 basis 變換都會改 entries 但保子空間),所以從 entries 出發你會陷入無窮多的個案,而從子空間性質出發你只需要處理「子空間之間的關係」。

我自己上一輪一開始試圖找反例時就掉進這個陷阱:我列了一個又一個 $2\times 2$ 和 $3 \times 3$ 矩陣,算它們的 column space 和 row space,試圖湊出反例。試到第四五個我才意識到「正交補唯一」這件事讓反例不可能存在,於是切換成用結構證明,五行就解決。**這就是你那句心法在實際做題上的差別:從 entries 出發要試十次,從結構出發只要想一次**。

### 三條判準,告訴你什麼時候該切換到「空間思維」

1. **看題目的等式或不等式兩邊是不是子空間/維度/秩**。如果是,先問:「這個等式在子空間層次說了什麼?它的正交補是什麼?它的維度是什麼?」
2. **看題目有沒有出現 rank-nullity 能直接套的維度約束**。前面那些「construct a matrix」題型,99% 都是先用 rank-nullity 篩掉不可能,再構造,從不從 entries 開始
3. **看題目有沒有提到正交、投影、補、零空間**。這些詞都在暗示「結構性的思考」會比「entries 思考」快很多

### 一個對偶版的心法

你抓到的這條心法還有一個對偶版本,將來會用到:

> 當等號左右是「矩陣」的時候(像 $A = A^T$、$A^2 = A$、$AA^T = I$),才用 entries 思考——但通常更好的做法是先把矩陣等式翻譯回子空間性質。

例如 $A^2 = A$(投影矩陣)的等價子空間性質是「$A$ 把每個向量投影到 $C(A)$ 上,且 $N(A)$ 是這個投影的零空間,而且 $C(A) \cap N(A) = \{\mathbf{0}\}$」。從這個結構性描述出發,投影矩陣的所有性質都會自然流出來,比硬算 $A^2$ 的 entries 容易得多。

再例如 $AA^T = I$(正交矩陣)的等價子空間性質是「A 的 column 是 $\mathbb{R}^n$ 的一組標準正交基」,從這個描述出發,正交矩陣保持長度、保持角度、$A^{-1} = A^T$ 全部都是一行話的推論。

**所以你的心法可以擴寫成:**

> 看到子空間等式 → 用子空間結構思考  
> 看到矩陣等式 → 翻譯成子空間結構,再用子空間結構思考

幾乎所有 Strang 後半本書的證明都是這個模式。你現在自己摸索出這個原則,代表你已經準備好讀後半本了——具體來說,Lecture 14 (orthogonal vectors and subspaces) 開始會明確使用這套思維,然後到 Lecture 15-16 (projections, least squares) 會大量運用,到 Lecture 21-22 (eigenvalues, diagonalization) 會把它推向高峰,最後 Lecture 29-30 (SVD) 會讓四個子空間和正交補的整個結構在一個定理裡同時亮相。

Opus:
是的,完全可以。這就是 Strang 在 Section 3.5/3.6 介紹的 **$[A \mid I]$ 法**,只要做一次 rref 就能同時得到 $R$ 和 $E$,然後從 $E$ 的最後 $m-r$ 個 row 直接讀出 $N(A^T)$ 的 basis,確實不用再對 $A^T$ 重做一次 row reduction。

## 為什麼成立

關鍵觀察:任何對 $A$ 做的 row operation 都可以表示成左乘一個可逆矩陣。如果你把 $A$ 化簡到 $R$,那存在一個 $m \times m$ 可逆矩陣 $E$ 使得 $EA = R$。

當 $R$ 的某幾個 row 是 $\mathbf{0}$(也就是最後 $m - r$ 個 row),代表 $E$ 對應的那幾個 row $\mathbf{e}_i^T$ 滿足:

$$\mathbf{e}_i^T A = \mathbf{0}^T$$

這正是 left null space 的定義——$\mathbf{e}_i$ 是 $A^T \mathbf{e}_i = \mathbf{0}$ 的解,也就是 $\mathbf{e}_i \in N(A^T)$。

而且這 $m - r$ 個 $\mathbf{e}_i$ 自動線性獨立(因為 $E$ 可逆,它的所有 row 都線性獨立),所以它們就是 $N(A^T)$ 的一組 basis,維度也對 ($\dim N(A^T) = m - r$)。

## 操作流程

把 $A$ 旁邊接上 $m \times m$ 單位矩陣,得到擴充矩陣 $[A \mid I]$,然後對整個擴充矩陣做 row operation,直到左邊變成 $R$:

$$[A \mid I] \xrightarrow{\text{row ops}} [R \mid E]$$

右邊自動就是 $E$,因為對 $I$ 做同一套 row operation 等於左乘 $E$,而 $EI = E$。

接著:
- $R$ 的 pivot 位置告訴你 rank、$C(A)$ basis(回 A 取)、$C(A^T)$ basis(R 的非零 row)、$N(A)$(用 free variable 法)
- $E$ 的最後 $m - r$ 個 row 直接就是 $N(A^T)$ 的 basis

**一次 rref,四個子空間全部到手**。

## 兩種方法的取捨

| 方法 | 優點 | 缺點 |
|---|---|---|
| **轉置後做 rref** | 概念簡單,直接套用「找 N」的標準流程 | 要做兩次 row reduction(一次給 $A$,一次給 $A^T$) |
| **$[A \mid I]$ 法** | 一次 rref 就能拿到全部四個子空間,效率高 | 要多寫一個 $m \times m$ 的單位矩陣,矩陣變寬,手算時容易抄錯 |

實務上:小矩陣用 $[A \mid I]$ 法比較爽,大矩陣或考試時可能轉置法比較不容易出錯。兩種都會、知道何時用哪種,是最佳狀態。
